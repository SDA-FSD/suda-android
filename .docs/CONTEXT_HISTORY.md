## 최근 작업 메모

- **DefaultToast 세로 패딩·minHeight 제거**: 세로 패딩 12 복원, `minHeight` 제거로 한 줄은 텍스트+패딩 높이만 사용·두 줄은 위아래 여백 확보. `lib/utils/default_toast.dart`, `.docs/CONTEXT.md`.
- **DefaultToast 가로 shrink-wrap**: `Center`가 유한 maxWidth에서 전폭으로 확장되던 문제를 피하기 위해 바깥·안쪽 모두 `Align(alignment: Alignment.center, widthFactor: 1, heightFactor: 1)`로 교체해 짧은 문구일 때 pill 가로가 콘텐츠에 맞게 줄어듦. `lib/utils/default_toast.dart`, `.docs/CONTEXT.md`.
- **DefaultToast 세로 패딩 제거**: pill 내부 세로 패딩(기존 12+12) 제거, `minHeight` 48이 곧 pill 높이. `lib/utils/default_toast.dart`, `.docs/CONTEXT.md` 공통 UI 유틸 항목.
- **Result V2 Feedback fallback**: `overallFeedback`가 null/빈 문자열이면 l10n `roleplayResultFeedbackInsufficientWords`를 대신 노출.
- **Result V2 Got it 복귀 전 동기화**: Got it 탭 시 Overview로 pop하기 전에 best-effort로 `GET /v1/users`로 `UserDto`를 최신화해 Main에 반영(`MainUserSync.notifyUserUpdated`)하고, `GET /v1/roleplays/{roleplayId}`로 Overview를 재조회한 뒤 `RoleplayStateService.setOverview`로 반영한다. Overview는 `RoleplayStateService.overviewUpdateTick`을 listen해 pop 복귀 시 별/상태가 갱신된다. 구현: `lib/screens/roleplay/result_v2.dart`, `lib/screens/roleplay/overview.dart`, `lib/services/roleplay_state_service.dart`, 문서: `.docs/CONTEXT_SCREEN.md` §17-1.
- **Ending → Result V2 전환**: Next 탭 시 풍선 확대·버튼 fade 애니메이션 제거. 별점 `PUT`은 기존처럼 비동기 발사 후 `RoleplayRouter.replaceWithResultV2()` 즉시 호출. 구현 `lib/screens/roleplay/ending.dart`, 문서 `.docs/CONTEXT_SCREEN.md` §14·§17-1, `.docs/CONTEXT_ROLEPLAY.md`.
- **Result V2 Expression 카드 메가폰·북마크**: 메가폰 `GET /v1/roleplays/results/{id}/expressions/{i}/sound`(`TtsResultDto`·Playing과 동일 CDN/bytes·`just_audio`), 로딩/재생 중 틴트 `#121212`, 타 카드 탭 시 이전 재생 중단, 완료·오류 시 `#0CABA8` 복귀만. 북마크 `POST /v1/users/expressions` `{roleplayResultId, expressionIndex}`, 200 시 `bookmark_on` + l10n `expressionSavedToProfile`, 이미 on이면 무시, 실패 시 HTTP 코드·간단 문구 토스트. V2 진입 시 북마크 UI는 전부 off. 구현·문서: `lib/screens/roleplay/result_v2.dart`, `RoleplayApi.getResultExpressionSound`, `UserApi.saveUserExpression`, `.docs/CONTEXT_SCREEN.md` §17-1, `.docs/CONTEXT_ROLEPLAY.md` §6-9.
- **Result V2 본문 타이밍·스타일**: 박스↔Feedback gap 20(스크롤 상단 패딩 반영). effect 완료 후 Feedback 즉시·Expression Upgrade 500ms 후·Got it!/Report 1s 후 동시 노출 + 240ms `easeOut` fade-in. Expression 카드 배경 `#80D7CF`, meaningUserLanguage 행 좌 패딩 +30.
- **Result V2 effect 이후 본문**: `LikeProgressEffect` 완료 후 `done` 제거. Feedback(슬라이드·overallFeedback 민트 박스)·Expression Upgrade(슬라이드·가로 카드 70%·IntrinsicHeight)·Got it!/Report(`#054544` 텍스트) 구현. 파일 `lib/screens/roleplay/result_v2.dart`.
- **Result V2 요약 카드 간격·Like 아이콘**: `lib/screens/roleplay/result_v2.dart` `_buildStatCard`에서 타이틀(Mission/Words/Like)과 둘째 줄 값 영역 사이 `SizedBox(height: 10)` 제거. Like 카드의 `like_at_result.png`는 30×30.
- **Result V2 1차 박스레이어 구현**: `lib/screens/roleplay/result_v2.dart`를 중앙 박스레이어 중심 구조로 재구성. 전체 배경은 상단 `#054544` → 하단 `#0CABA8` 그라데이션 유지. 박스레이어는 별 행 위에 레거시 `result.dart`와 동일한 `SizedBox(height: 30)` 여백 후 별 3개 + mainTitle + 민트색 subTitle + Mission/Words/Like 3카드로 구성하며, Like 카드는 `assets/images/like_at_result.png` 24×24와 기존 Result와 동일한 그라데이션 숫자를 사용한다.
- **Result V2 진입 후 타이밍**: bottom-up 라우트로 화면이 fully shown 된 시점을 `ModalRoute.animation` 완료로 감지하고, 1초 뒤 박스레이어를 상단 영역으로 이동시키면서 동시에 `LikeProgressEffect.play()`를 실행한다. effect 파라미터는 `beforeLikePoint`, `afterLikePoint`, `beforeLevel`, `afterLevel`, `beforeProgressPercentage`, `afterProgressPercentage`를 사용한다. effect 완료 후에는 Feedback·Expression Upgrade·Got it!·Report 본문 레이어를 노출한다(상세 `.docs/CONTEXT_SCREEN.md` §17-1).
- **RoleplayResultDto V2 필드 확장**: `lib/models/roleplay_models.dart`의 `RoleplayResultDto`에 `version`, `overallFeedback`, `expressionUpgrades(List<ExpressionUpgradeDto>)` 추가. `ExpressionUpgradeDto`는 `expression`, `meaningUserLanguage`, `rephrasedSentence` 필드를 가진다. Profile 히스토리 목록 DTO `RpSimpleResultDto`에도 `version` 필드 추가.
- **Profile 히스토리 version 분기**: `lib/screens/profile.dart`에서 RP 이력 썸네일 탭 시 `item.version == 2`면 `lib/screens/roleplay/history_v2.dart`의 `HistoryScreenV2`로, `version == 1` 또는 예외값(null/기타)이면 기존 `lib/screens/roleplay/history.dart`의 `HistoryScreen`으로 진입하도록 변경. `history_v2`는 현재 라우팅 가능한 최소 스켈레톤만 준비하고 실제 UI는 후속 지침에서 구현한다.
- **History V2 구현**: `lib/screens/roleplay/history_v2.dart`를 Result V2 기준 히스토리 화면으로 구현. resultId로 `getRoleplayResult` 재조회, View Chat/View Ending 동작 유지, Expression 북마크는 `savedExpressionIndexes`를 초기 on으로 반영하고 OFF↔ON(POST/DELETE 쿼리) 토글 지원.
- History V2 상단 박스는 Result V2와 동일하게 별·mainTitle·subTitle 아래 Mission/Words/Like 3카드를 함께 노출한다.
- **Result 기본 플로우를 Result V2로 교체**: `lib/screens/roleplay/result.dart`는 레거시로 유지하고, 실제 종료 플로우의 Result 연결은 `lib/screens/roleplay/result_v2.dart`로 전환. Playing의 resultId>0 비완수 분기와 Ending의 Next 완료 분기에서 모두 `lib/routes/roleplay_router.dart`의 `replaceWithResultV2()`를 사용한다.
